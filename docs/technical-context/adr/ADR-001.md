# ADR-001: Uso do Llama via NVIDIA NIM como LLM Principal

**Data:** 2026-03-15
**Status:** Aceito
**Autores:** Time Jarvis

---

## Contexto

O projeto Jarvis precisa de um LLM capaz de:
- Gerar texto em português com qualidade adequada
- Ser executado sem custo inicial significativo
- Suportar prompts longos (contexto RAG com múltiplos chunks)
- Ser acessível via API padrão OpenAI-compatible

Foram avaliadas as seguintes opções:
1. OpenAI GPT-4 / GPT-3.5 — pago, custo variável por token
2. Anthropic Claude — pago, excelente qualidade mas custo elevado
3. Google Gemini — opção gratuita limitada
4. **Llama 3.x via NVIDIA NIM API** — gratuito na camada de desenvolvimento, open-source, OpenAI-compatible

---

## Decisão

Adotar **Llama 3.1 70B Instruct** disponibilizado via **NVIDIA NIM API** como LLM principal do Jarvis.

Endpoint: `https://integrate.api.nvidia.com/v1`
Modelo: `meta/llama-3.1-70b-instruct`

A integração será feita via LangChain usando o conector OpenAI-compatible, apontando para a base URL da NVIDIA.

---

## Consequências

### Positivas
- **Sem custo inicial:** NVIDIA oferece créditos gratuitos para desenvolvimento
- **Open-source:** modelo Llama é open-source (Meta AI), sem lock-in de licença de modelo
- **Compatibilidade:** API OpenAI-compatible facilita migração futura
- **Qualidade:** Llama 3.1 70B tem bom desempenho em português

### Negativas / Riscos
- **Dependência da NVIDIA:** infraestrutura de inferência fora do nosso controle
- **Limites de rate:** API gratuita tem limites de requisições
- **Latência:** inferência remota pode ser mais lenta que local
- **Mudanças de API:** NVIDIA pode alterar preços ou disponibilidade

### Mitigações
- Abstrair o LLM via interface LangChain para facilitar troca futura
- Monitorar uso para não exceder limites gratuitos
- Documentar modelos alternativos em ADR futura se necessário

---

## Alternativas Descartadas

| Opção | Motivo da Rejeição |
|---|---|
| OpenAI GPT-4 | Custo elevado para fase inicial |
| Anthropic Claude | Custo elevado, sem tier gratuito adequado |
| Ollama local | Hardware do time insuficiente para 70B local |
| HuggingFace Inference API | Menor qualidade no tier gratuito |
